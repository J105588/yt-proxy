# YT Proxy (YouTube / Niconico Proxy)

YouTubeおよびニコニコ動画の動画情報を取得・中継し、Google Apps Script (GAS) などの外部システムと連携して再生するためのプロキシサーバーである。

## 必要条件

本システムを構築・実行するためには、以下のソフトウェアおよびバイナリが必要となる。

1. **Node.js** (v18以降推奨)
   * サーバー本体の実行環境として使用する。
2. **yt-dlp**
   * 動画情報の取得およびストリーミングURLの抽出に使用する。
   * インストール後、システム環境変数 `PATH` に登録し、コマンドプロンプト等から `yt-dlp` コマンドを直接実行できる状態にする必要がある。
3. **FFmpeg**
   * 動画の中継や処理に使用する。
   * インストール後、システム環境変数 `PATH` に登録し、`ffmpeg` コマンドを実行できる状態にする必要がある。
4. **cloudflared**
   * 外部公開用のトンネル（Cloudflare Tunnel）を開設するために使用する。
   * Windows環境の場合、`cloudflared.exe` を本リポジトリのルートディレクトリ直下に配置する。

## クリーン環境からの構築手順

新規環境に本システムを構築する手順は以下の通りである。

### 1. リポジトリの準備
本リポジトリをローカル環境にクローンまたはダウンロードし、ディレクトリに移動する。
```bash
git clone https://github.com/J105588/yt-proxy.git
cd yt-proxy
```

### 2. 依存パッケージのインストール
Node.jsのパッケージマネージャーを使用して、必要なライブラリをインストールする。本リポジトリのルートディレクトリで以下のコマンドを実行する。
```bash
npm install
```

### 3. 環境設定（.envファイルの作成）
ルートディレクトリにある `.env.example` をコピーして `.env` ファイルを作成する。
```bash
copy .env.example .env
```
作成した `.env` ファイルをテキストエディタで開き、各項目を設定する。

* `PORT`: プロキシサーバーが待機するポート番号（デフォルト: `3000`）
* `GAS_URL`: 連携先 Google Apps Script (GAS) のWebアプリデプロイURL
* `GAS_KEY`: GAS側と認証を行うためのセキュリティキー（任意の文字列）

### 4. クッキーの設定（ニコニコ動画の制限動画用）
ニコニコ動画のログイン制限（年齢制限やコミュニティ限定など）がかかった動画を視聴する場合、ブラウザから Netscape 形式でクッキーをエクスポートし、ファイル名を **`cookies.txt`** としてルートディレクトリ直下に配置する。

### 5. cloudflaredの配置
Cloudflareの公式サイト等から環境に応じた `cloudflared` のバイナリをダウンロードする。Windows環境の場合は `cloudflared.exe` をダウンロードし、ルートディレクトリ直下に配置する。

## 使い方

システムの制御は、用意されているバッチファイルを用いて行う。

### 起動
```bash
start.bat
```
* **動作内容**: 
  1. 指定されたポート（デフォルト: `3000`）を使用している古いプロセスを自動でクリーンアップする。
  2. `server.js` を起動する。
  3. `cloudflared.exe` を使用してローカルサーバーへのトンネル（パブリックURL）を開設する。
  4. 開設されたトンネルのURLを `.env` で設定した `GAS_URL` へ自動で通知（POST送信）する。
  5. 起動後は監視モードに入り、サーバーやトンネルの死活監視を行う（4時間ごとに自動再起動、フリーズ検知時の自動復旧を実行する）。

### 停止
```bash
stop.bat
```
* **動作内容**: 起動中の `server.js`（Node.jsプロセス）、`cloudflared.exe`、および `ffmpeg.exe` を強制終了し、クリーンアップを行う。

### 再起動
```bash
restart.bat
```
* **動作内容**: 実行中のシステムを一度停止し、再度初期化から起動処理を行う。
